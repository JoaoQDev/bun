## Codebase Patterns
- C++ source files are NOT added directly to cmake/sources/CxxSources.txt — that file is auto-generated by `scripts/glob-sources.mjs` from glob patterns in `cmake/Sources.json`. Add new directory globs there instead.
- On macOS, symbols referenced from Zig that are defined in C++ must be added to `src/symbols.txt` (exported_symbols_list) or linking will fail with "Undefined symbols".
- Bun's `Output.pretty()` uses `<tag>` syntax for colors (e.g. `<b>`, `<green>`, `<r>`). Any `<word>` in the format string is interpreted as a color tag — avoid angle brackets in help text unless they're valid color names.
- CLI commands are registered in 4 places in `src/cli.zig`: (1) import at top, (2) Tag enum variant, (3) `which()` string matcher, (4) `start()` switch handler, (5) `char()` for crash reports. The `RootCommandMatcher` is `ExactSizeMatcher(12)` — command names must be <= 12 chars.
- Simple CLI commands that don't need Bun's full context initialization can take just `allocator` (like DiscordCommand), avoiding `Command.init()`.
- `Zig__GlobalObject__create` creates both VM and GlobalObject; it requires Zig runtime init (Bun__getVM, etc.). Cannot create GlobalObjects from standalone C++ without Bun's infrastructure.
- JSC module loading: `JSC::loadAndEvaluateModule()` returns `JSInternalPromise*`; default export via `vm.propertyNames->defaultKeyword` on module namespace.
- JSC function calls from C++: use `JSC::call()` or `JSC::profiledCall()` with `MarkedArgumentBuffer`.
- New .cpp files in `src/serverless/` are auto-discovered by the build (glob in cmake/Sources.json). No need to manually register them.
- Use pimpl idiom in serverless C++ headers to avoid leaking JSC includes to consumers (e.g., TenantManager).
- Commands that need JSC must go through `Command.init()` + `VirtualMachine.init()` + `holdAPILock()`. Follow the REPL command pattern (repl_command.zig) for initialization sequence.
- `uses_global_options` in cli.zig Tag must be set to false for commands that parse their own args (otherwise Bun's global arg parser intercepts --help, etc.).
- `JSC::loadAndEvaluateModule()` resolves to undefined — use `globalObject->moduleLoader()->importModule()` to get a promise that resolves to the module namespace object.
- `ScriptExecutionContextIdentifier` is in the `Bun::` namespace (not global) — use `Bun::ScriptExecutionContextIdentifier`.
- Include `<JavaScriptCore/Completion.h>` for `JSC::loadAndEvaluateModule`. Include `<JavaScriptCore/SourceOrigin.h>` for `JSC::SourceOrigin`.
- Use `DECLARE_TOP_EXCEPTION_SCOPE` (not `DECLARE_CATCH_SCOPE` which doesn't exist in this WebKit version).
